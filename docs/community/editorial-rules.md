Эти правила нужны, чтобы книга оставалась цельной после правок разных авторов.

## Язык

Пишите ясным деловым русским языком. Английские термины оставляйте там, где они являются названием технологии, продукта, API или устойчивым термином индустрии.

Термины `AI-Native`, `skill`, `source of truth`, `MCP` и `human-in-control` используются как элементы методологии. При добавлении новых терминов сначала проверьте, есть ли они в словаре.

## Структура

Одна страница должна решать одну задачу: объяснить понятие, разобрать ошибку, описать архитектурный элемент или дать практический шаблон.

Если правка добавляет новый крупный смысл, лучше создать отдельный подраздел, а не перегружать существующий абзац.

## Проверка

Перед pull request проверьте:

1. Не сломалась ли навигация в `sidebars.ts`.
2. Не появились ли противоречия с соседними главами.
3. Не изменился ли смысл термина без обновления словаря.
4. Проходит ли `npm run build`.

## Тон

Текст должен помогать руководителю принять решение или собрать рабочий контур. Избегайте рекламных усилителей, тяжёлого канцелярита и фраз, которые звучат убедительно, но не добавляют проверяемого смысла.
