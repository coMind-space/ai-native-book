## 3. Правила при конфликте источников

Опишите, что делать, если данные расходятся.

| Конфликт | Главный источник | Кто решает | Что делает ассистент |
| --- | --- | --- | --- |
| CRM и документ расходятся по статусу клиента |  |  |  |
| Трекер и устная договоренность расходятся по сроку |  |  |  |
| Шаблон в папке отличается от шаблона в базе знаний |  |  |  |
| Память проекта противоречит актуальному документу |  |  |  |

Базовое правило:

Если память, переписка или устная договоренность противоречат source of truth, ассистент должен остановиться, показать расхождение и запросить подтверждение.

## 7. Минимальное правило для ассистента

Перед подготовкой результата ассистент обязан:

1. Определить нужные источники.
2. Проверить доступ.
3. Получить актуальные данные.
4. Отметить расхождения.
5. Не использовать память как замену source of truth.
6. Не передавать чувствительные данные в LLM без правил обезличивания.
